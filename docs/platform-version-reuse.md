# Platform Version Reuse Guide

This document summarizes tested source formats and how to reuse the parsers for future figure skating result platforms.

## Tested Source Profiles

### `old_isucalcfs`

Used for older ISUCalcFS-generated result sites.

Validated example:

- `https://cup.clubdenkovastaviski.com/2013/ISU/index.htm`

Known page patterns:

- `index.htm`
- `CAT###EN.HTM`
- `CAT###RS.HTM`
- `SEG###.HTM`
- `SEG###OF.HTM`
- `*_Scores.pdf`

Characteristics:

- malformed older table HTML.
- uppercase `.HTM` links.
- generated by `ISUCalcFS`.
- officials links usually labelled `Officials`.
- result-detail links usually labelled `Starting Order / Result Details`.
- judges-score PDFs are Crystal Reports style.

Current parser coverage:

- event metadata.
- categories and segments.
- entries.
- category results.
- segment result details.
- officials and role grouping.
- judges-score PDF summary rows.

### `fs_manager`

Used for newer Swiss Timing FS Manager result sites.

Validated examples:

- `https://cup.clubdenkovastaviski.com/2024/ISU/index.htm`
- `https://cup.clubdenkovastaviski.com/2024/NonISU/index.htm`
- `https://sofia-trophy.clubdenkovastaviski.com/2026/ISU/`
- `https://www.bsf.bg/figure-skating/ice-peak-trophy/18-19.04.2026/index.htm`

Known page patterns:

- `index.htm` or directory URL ending in `/`
- `CAT###EN.htm`
- `CAT###RS.htm`
- `SEG###.htm`
- `SEG###OF.htm`
- `*_JudgesDetailsperSkater.pdf`
- optional event protocol PDF

Characteristics:

- generated by `FS Manager by Swiss Timing, Ltd.`
- lowercase `.htm` links.
- officials links labelled `Panel of Judges`.
- segment details labelled `Starting Order / Detailed Classification`.
- can include ISU bio links.
- can include `Club` column for local/federation events.
- PDF layout supports detailed element and judge-mark parsing.

Current parser coverage:

- event metadata.
- source context: host, event path, ISU/NonISU stream when present.
- categories and segments.
- schedule.
- event protocol PDF link.
- entries.
- category results.
- segment result details.
- officials and judge numbers.
- ISU bio URL and source skater id when real.
- club/nation representation controlled by config.
- judges-score PDF summary rows.
- PDF executed element rows.
- PDF per-judge GOE marks.
- PDF program component judge marks.
- PDF deductions detail.
- PDF bonus values when present.
- structured element markers.

## Source-Profile Settings

Settings live in:

- `config/source_profiles.json`

The config controls source behavior that should not be hardcoded in parsers.

Current settings include:

- profile id.
- host match.
- event path prefix match.
- parser profile.
- competition stream.
- representation behavior.

Example representation behavior:

- ISU/international events: primary representation is `nation`.
- Ice Peak Trophy local event: primary representation is `club`, while still storing `nation`.

Recommended rule:

- Add a new config profile before changing parser code.
- Only change parser code if the HTML/PDF structure is actually different.

## Reusing For A New Platform Version

When a new platform URL is provided:

1. Fetch the index page.
2. Check generator metadata:
   - `ISUCalcFS` means likely `old_isucalcfs`.
   - `FS Manager by Swiss Timing, Ltd.` means likely `fs_manager`.
3. Run the matching parser:

```bash
python3 scripts/parse_old_isu.py <url> --validate
python3 scripts/parse_fs_manager.py <url> --validate
```

4. Inspect validation issues.
5. Fetch one sample each:
   - entries page.
   - category result page.
   - segment result page.
   - officials page.
   - judges-score PDF.
6. Check whether entries/results use:
   - `Nation` only.
   - `Club` plus `Nation`.
   - another representation column.
7. Add or update `config/source_profiles.json`.
8. Parse sample PDF:

```bash
python3 scripts/parse_judges_pdf.py <pdf-url-or-path> --validate
```

9. Compare counts:
   - category entries count.
   - category result count.
   - segment result count.
   - PDF skater count.
   - officials judge count vs PDF judge columns.

## Data To Preserve For Reimport

For every import, preserve:

- source profile id.
- parser profile.
- root URL.
- document URL.
- document type.
- fetched timestamp.
- content hash.
- raw extracted text for PDFs where possible.
- import run id.
- parse issues.
- source row order.
- raw source names.
- source context.

For skaters, preserve source appearances separately from canonical profiles:

- raw name.
- normalized name.
- nation.
- club.
- representation type.
- representation value.
- bio URL.
- source skater id when real.
- category.
- segment.
- event.
- source URL.

## Identity Notes

Do not merge skaters only by name.

Preferred early matching priority:

1. Real source skater id, for example `isufs00107843`.
2. Normalized full name + nation + event/source context.
3. Normalized full name + club + event/source context for local club events.
4. Manual/confirmed identity match.

Placeholder bio ids such as `isufs00000bul` are not real skater ids and should not be used for identity matching.

## Officials And Judges

Officials are segment assignments, not only global people.

Store:

- function text.
- normalized role group.
- judge number.
- name.
- nation.
- category.
- segment.
- source URL.

Known role groups:

- `referee`
- `technical_panel`
- `judge`
- `event_operations`
- `other`

PDF `J1`, `J2`, etc. should map to `Judge No.1`, `Judge No.2`, etc. from the officials page.

## PDF Notes

FS Manager PDF parser now handles:

- compact score strings such as `82.9745.24`.
- compact PCS/deduction strings such as `12.810.00`.
- negative deduction strings such as `32.00-1.00`.
- element markers:
  - `!`
  - `<`
  - `<<`
  - `q`
  - `e`
  - `F`
  - `x`
  - `b`
  - `*`
  - `REP`
- missing judge marks represented as `-`, stored as `None`.
- optional bonus column.
- variable judge counts.

For each PDF element row, store:

- raw element text.
- source element code.
- marker-stripped base element code.
- markers.
- info text.
- base value.
- GOE.
- judge scores.
- bonus.
- panel score.
